КАК ОПУБЛИКОВАТЬ САЙТ И СОБИРАТЬ ОТВЕТЫ

Сайт статический, ему не нужен сервер. Публикуется на GitHub Pages бесплатно.
Ответы участников собираются в вашу Google-таблицу. Ниже два раздела по порядку.


РАЗДЕЛ 1. ТАБЛИЦА ДЛЯ ОТВЕТОВ

1. Зайдите на drive.google.com под своим аккаунтом. Создайте новую таблицу
   (Создать, дальше Google Таблицы). Назовите как удобно, например «Ответы ППМ».
2. В верхнем меню откройте Расширения, затем Apps Script. Откроется редактор кода.
3. Удалите всё, что в редакторе уже есть, и вставьте код из раздела «Код для вставки»
   (он ниже, между линиями).
4. Нажмите значок сохранения (дискета).
5. Нажмите синюю кнопку «Развернуть», затем «Новое развёртывание».
6. Нажмите на шестерёнку рядом со словом «Тип» и выберите «Веб-приложение».
7. «Запуск от имени» оставьте «Я». В поле «У кого есть доступ» выберите «Все».
8. Нажмите «Развернуть». Google попросит разрешение. Согласитесь
   (Дополнительные настройки, потом «Перейти к проекту», потом «Разрешить»).
9. Скопируйте показанный адрес веб-приложения. Он длинный и заканчивается на /exec.
10. Откройте файл config.js из папки сайта обычным Блокнотом. Вставьте скопированный
    адрес между кавычками в строке window.PPM_SUBMIT_URL. Сохраните файл.

После этого каждый ответ участника будет добавляться новой строкой в вашу таблицу.


КОД ДЛЯ ВСТАВКИ
--------------------------------------------------------------------
function doPost(e) {
  var lock = LockService.getScriptLock();
  lock.waitLock(30000);
  try {
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheets()[0];
    if (sheet.getLastRow() === 0) {
      sheet.appendRow(['Время', 'Код', 'Инструмент', 'Этап', 'Возраст', 'Ответы', 'Служебное']);
    }
    var d = JSON.parse(e.postData.contents);
    sheet.appendRow([new Date(), d.code || '', d.instrument || '', d.stage || '', d.age || '', d.payload || '', d.meta || '']);
    return ContentService.createTextOutput('OK');
  } catch (err) {
    return ContentService.createTextOutput('ERR ' + err);
  } finally {
    lock.releaseLock();
  }
}

РАЗДЕЛ 2. ПУБЛИКАЦИЯ САЙТА НА GITHUB PAGES

1. Заведите бесплатный аккаунт на github.com, если его ещё нет.
2. Создайте новый репозиторий (кнопка «New» или «New repository»). Имя любое
   латиницей, например ppm-test. Тип «Public». Нажмите «Create repository».
3. На странице репозитория нажмите «Add file», затем «Upload files».
4. Перетащите туда ВСЕ файлы из папки сайта. Это index.html, styles.css, app.js,
   config.js и три файла, имена которых начинаются на data. Перетаскивайте сами файлы,
   а не папку целиком.
5. Нажмите «Commit changes».
6. Откройте «Settings» репозитория, в левом столбце выберите «Pages».
7. В разделе «Branch» выберите ветку main и папку / (root). Нажмите «Save».
8. Подождите минуту и обновите страницу. Вверху появится адрес вида
   https://ваш-логин.github.io/ppm-test/. Это и есть ссылка для участников.


ВАЖНОЕ

Файл config.js с вашим адресом должен попасть на GitHub вместе с остальными. Если вы
вписали адрес уже после загрузки, просто загрузите обновлённый config.js заново тем же
способом, поверх старого.

После настройки сделайте одно пробное прохождение и проверьте, что в таблице появилась
строка. Если строки нет, откройте развёртывание заново и убедитесь, что в поле «У кого
есть доступ» выбрано «Все», а скопированный адрес заканчивается на /exec.

Если адрес в config.js пустой, сайт всё равно работает, но ответы не уходят сами. Тогда
участник в конце нажимает «Скопировать ответы» или «Скачать файл» и присылает их вам.


КАК СВЯЗЫВАЮТСЯ ОТВЕТЫ ДО И ПОСЛЕ

Каждый участник придумывает короткий код и запоминает его. Первичная и вторичная
диагностика уходят с этим кодом, поэтому в таблице вы сопоставите ответы до и после по
совпадению кода. Напомните участникам записать код и не менять его к концу программы.
